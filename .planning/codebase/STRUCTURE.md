# Codebase Structure

**Analysis Date:** 2026-03-05

## Directory Layout

```text
pushstr/
├── src/                         # Browser extension source (runtime + UI pages)
├── mobile/                      # Flutter mobile app, platform projects, generated bridge
├── pushstr_rust/                # Rust core library used by mobile via flutter_rust_bridge
├── wasm_crypto/                 # Rust->WASM crypto module used by extension
├── scripts/                     # Extension build/package/patch scripts
├── documentation/               # Product and functional documentation
├── dist/                        # Built extension artifacts (generated output)
├── types/                       # Local TypeScript type roots
├── test/                        # Lightweight test notes
├── vendor/                      # Bundled third-party browser assets
├── manifest.json                # Firefox/MV2 extension manifest
├── manifest.chrome.json         # Chrome/MV3 extension manifest
├── package.json                 # Node scripts/dependencies for extension tooling
├── flutter_rust_bridge.yaml     # Rust<->Dart bridge generation config
└── README.md                    # Top-level project overview
```

## Directory Purposes

**`src/`:**
- Purpose: Extension runtime and UI implementation.
- Contains: Background orchestration, popup/options scripts, HTML/CSS assets, WASM glue (`wasm_crypto.js`, `wasm_crypto_bg.wasm`).
- Key files: `src/background.js`, `src/popup.js`, `src/options.js`, `src/background.html`, `src/popup.html`, `src/options.html`.
- Subdirectories: Flat layout (no nested feature directories).

**`mobile/`:**
- Purpose: Flutter application and platform integration.
- Contains: App UI/state, sync workers, local notification setup, generated FFI bridge, Android/iOS projects.
- Key files: `mobile/lib/main.dart`, `mobile/lib/sync/sync_controller.dart`, `mobile/lib/sync/rust_sync_worker.dart`, `mobile/lib/notifications.dart`, `mobile/pubspec.yaml`.
- Subdirectories: `mobile/lib/`, `mobile/android/`, `mobile/ios/`, `mobile/test/`, `mobile/tools/`.

**`mobile/lib/bridge_generated.dart/`:**
- Purpose: Generated Dart bindings for Rust API.
- Contains: Generated bridge APIs and runtime glue.
- Key files: `mobile/lib/bridge_generated.dart/api.dart`, `mobile/lib/bridge_generated.dart/frb_generated.dart`, `mobile/lib/bridge_generated.dart/frb_generated.io.dart`.
- Subdirectories: None (generated files only).

**`pushstr_rust/`:**
- Purpose: Rust shared core for relay, DM, giftwrap, and media operations.
- Contains: FRB-exported API, generated Rust bridge code, cargo config, build outputs.
- Key files: `pushstr_rust/src/api.rs`, `pushstr_rust/src/lib.rs`, `pushstr_rust/Cargo.toml`.
- Subdirectories: `pushstr_rust/src/`, `pushstr_rust/target/`.

**`wasm_crypto/`:**
- Purpose: Rust cryptography module compiled to WASM for browser extension use.
- Contains: AES-GCM, SHA-256, NIP-44 helpers.
- Key files: `wasm_crypto/src/lib.rs`, `wasm_crypto/Cargo.toml`.
- Subdirectories: `wasm_crypto/src/`.

**`scripts/`:**
- Purpose: JS build automation for the extension package.
- Contains: Esbuild bundling, zip packaging, WASM glue patching, ad-hoc DM test script.
- Key files: `scripts/build.js`, `scripts/package.js`, `scripts/patch_wasm_crypto.js`, `scripts/js_dm_test.js`.
- Subdirectories: None.

**`documentation/`:**
- Purpose: Product-level specs and changelog material.
- Contains: Functional specification and project notes.
- Key files: `documentation/pushstr_fsd.md`, `documentation/changelog.md`.
- Subdirectories: None.

**`types/`:**
- Purpose: Local type roots for TypeScript tooling.
- Contains: Ambient types for browser extension and Node toolchain.
- Key files: `types/firefox-webext-browser/index.d.ts`, `types/node/index.d.ts`.
- Subdirectories: `types/firefox-webext-browser/`, `types/node/`.

**`dist/`:**
- Purpose: Generated browser extension build output.
- Contains: Bundled JS, copied static assets, vendor files, WASM artifacts.
- Key files: `dist/background.js`, `dist/popup.js`, `dist/options.js` (generated by `scripts/build.js`).
- Subdirectories: `dist/vendor/`.

## Key File Locations

**Entry Points:**
- `manifest.json`: Firefox/MV2 entry wiring for background page + popup/options pages.
- `manifest.chrome.json`: Chrome/MV3 entry wiring for service worker + popup/options pages.
- `src/background.js`: Extension runtime entry and message command dispatcher.
- `src/popup.js`: Extension chat UI entry.
- `src/options.js`: Extension settings UI entry.
- `mobile/lib/main.dart`: Mobile app bootstrap entry (Flutter + Rust init).
- `pushstr_rust/src/api.rs`: Rust API entry surface consumed by Flutter bridge.

**Configuration:**
- `package.json`: Node scripts and extension dependency set.
- `tsconfig.json`: TypeScript compiler/type-root configuration.
- `vitest.config.ts`: Unit test config (expects `tests/` paths).
- `playwright.config.ts`: E2E test config (expects `tests/e2e/`).
- `flutter_rust_bridge.yaml`: FRB codegen routing (`rust_input`, `dart_output`).
- `pushstr_rust/Cargo.toml`: Rust core dependency/runtime config.
- `wasm_crypto/Cargo.toml`: WASM crypto dependency/runtime config.
- `mobile/pubspec.yaml`: Flutter app package/dependency config.

**Core Logic:**
- `src/background.js`: Relay connections, DM send/receive, key/profile storage, media upload/decrypt.
- `src/popup.js`: Conversation rendering, attachments, optimistic send UX.
- `src/options.js`: Profile/relay/contact management UI.
- `mobile/lib/main.dart`: Mobile messaging UI and interaction orchestration.
- `mobile/lib/sync/sync_controller.dart`: Bounded background sync + notification policy.
- `mobile/lib/sync/rust_sync_worker.dart`: Isolate-safe wrappers over Rust calls.
- `pushstr_rust/src/api.rs`: Relay+crypto core operations and media pipeline.
- `wasm_crypto/src/lib.rs`: Browser-side crypto primitives shared with background runtime.

**Testing:**
- `mobile/test/widget_test.dart`: Flutter widget scaffold test.
- `mobile/tools/ffi_dm_test.dart`: Manual/integration-oriented FFI DM check.
- `test/test.md`: Lightweight project test notes.
- `playwright.config.ts` and `vitest.config.ts`: Configured test harnesses (paths currently configured to `tests/`).

**Documentation:**
- `README.md`: Unified project overview and build/run instructions.
- `documentation/pushstr_fsd.md`: Functional specification.
- `MOBILE_APP_GUIDE.md` and `MOBILE_APP_SUMMARY.md`: Mobile-focused references.

## Naming Conventions

**Files:**
- Extension runtime and page scripts use lowercase role names: `src/background.js`, `src/popup.js`, `src/options.js`.
- Dart files use snake_case filenames: `mobile/lib/main.dart`, `mobile/lib/sync/sync_controller.dart`.
- Rust source uses snake_case module/file names: `pushstr_rust/src/api.rs`, `wasm_crypto/src/lib.rs`.
- Generated bridge files use `frb_generated*` suffixes: `mobile/lib/bridge_generated.dart/frb_generated.dart`, `pushstr_rust/src/frb_generated.rs`.
- Top-level product docs use uppercase or title names: `README.md`, `CHANGELOG.md`, `MOBILE_APP_GUIDE.md`.

**Directories:**
- Feature/runtime directories are short lowercase names: `src/`, `mobile/`, `scripts/`, `vendor/`.
- Language-specific ownership is directory-based: extension in `src/`, Flutter in `mobile/lib/`, Rust in `pushstr_rust/src/` and `wasm_crypto/src/`.
- Generated/build directories are clearly separated: `dist/`, `pushstr_rust/target/`, `mobile/.dart_tool/`.

**Special Patterns:**
- Browser runtime messages use kebab-case command strings in `src/background.js` and callers (`'get-state'`, `'send-gift'`, `'upload-blossom'`).
- Profile-scoped storage keys encode active identity suffixes in mobile (`pending_dms_<nsec>`, `contacts_<nsec>` in `mobile/lib/main.dart` / `mobile/lib/sync/sync_controller.dart`).
- Both Rust and JS use explicit `seq` tag handling for message ordering in `pushstr_rust/src/api.rs` and `src/background.js`.

## Where to Add New Code

**New Browser Extension Messaging Feature:**
- Primary code: `src/background.js` (command handling + relay/crypto orchestration).
- UI behavior: `src/popup.js` and related markup in `src/popup.html` + `src/popup.css`.
- Settings surface: `src/options.js` + `src/options.html` + `src/options.css`.
- Build wiring: `scripts/build.js` (if new static assets or entry changes are needed).

**New Mobile Messaging Behavior:**
- UI + state behavior: `mobile/lib/main.dart`.
- Background sync policy: `mobile/lib/sync/sync_controller.dart`.
- Isolate-safe Rust calls: `mobile/lib/sync/rust_sync_worker.dart`.
- Platform notification behavior: `mobile/lib/notifications.dart`.

**New Shared Crypto/Relay Capability:**
- Rust implementation: `pushstr_rust/src/api.rs`.
- Dart bridge exposure: regenerate files under `mobile/lib/bridge_generated.dart/`.
- Extension parity (if needed): `wasm_crypto/src/lib.rs` plus consumption in `src/background.js`.

**New Build/Packaging Automation:**
- Extension build pipeline: `scripts/build.js` and `scripts/package.js`.
- WASM glue hardening adjustments: `scripts/patch_wasm_crypto.js`.

**Testing Additions:**
- Flutter UI tests: `mobile/test/`.
- Browser tests: add suites under paths expected by `vitest.config.ts` and `playwright.config.ts`.
- Manual/FFI checks: `mobile/tools/`.

## Special Directories

**`dist/`:**
- Purpose: Bundled extension output.
- Source: Generated by `node scripts/build.js`.
- Committed: Yes currently; treat as build artifact.

**`pushstr_rust/target/`:**
- Purpose: Cargo build outputs (native libs and intermediates).
- Source: Generated by Rust build commands.
- Committed: Present in repo; treat as generated artifact.

**`mobile/.dart_tool/`:**
- Purpose: Flutter tool state and package graph outputs.
- Source: Generated by Flutter tooling.
- Committed: Present in repo snapshot; treat as generated artifact.

**Generated bridge files (dual-side):**
- Purpose: Rust<->Dart FFI glue.
- Source: Generated via `flutter_rust_bridge_codegen` using `flutter_rust_bridge.yaml`.
- Committed: Yes (`mobile/lib/bridge_generated.dart/*`, `pushstr_rust/src/frb_generated.rs`, and `pushstr_rust/src/src/frb_generated.rs`).
